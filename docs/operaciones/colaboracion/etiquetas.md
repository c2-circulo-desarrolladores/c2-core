# Etiquetas

Las etiquetas clasifican issues por prioridad y scope. 

Los issues pueden abrirse sin etiquetas, pero le dan valor agregado, en tanto ayudan a organizar el trabajo y definen áreas de responsabilidad.

Los owners deben crear etiquetas de scope según su repositorio (por ejemplo: `frontend`, `backend`, `docs`).

En C2, todo repositorio debería tener las siguientes etiquetas obligatorias:

**Prioridad:**

- `P1` — alta prioridad, atender pronto.
- `P2` — prioridad media.
- `P3` — prioridad baja, se puede diferir.
