# GitHub

## Workflow de GitHub

- Abrir issues para documentar cambios por hacer.
- Abrir Pull Requests para mergear ramas a `main` o a otras ramas.
- Incluir en el mensaje de la PR qué issues se cerrarían al mergear (ejemplo: Closes #10)

Abrir un issue — en tu repositorio o en el de otro — es la forma más directa de contribuir: registra un problema, propone una mejora o inicia una conversación.

Un issue abierto es una invitación a colaborar, no una crítica. Abre uno tan pronto identifiques algo que merezca atención: una feature, un fix, un bug, un cambio de estilo, una propuesta de refactorización, etc.
